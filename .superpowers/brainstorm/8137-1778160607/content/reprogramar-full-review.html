03 — ¿Qué pasa al reprogramar 10 pedidos a la vez?

Seleccioná la estrategia de fecha para bulk

X

Una fecha para todos

El modal pide una sola fecha/hora. Los N pedidos seleccionados quedan con el mismo callbackAt. Se informa cuántos pedidos se afectan.

✓ Simple ✓ Rápido ✗ Inflexible
Y

Fecha individual por pedido

El modal muestra una lista de pedidos y cada uno tiene su propio date picker. Muy granular pero complejo para muchos pedidos.

✓ Flexible ✗ UX pesada ✗ Escala mal
Z

Fecha default automática ⭐

No pide fecha. Asigna automáticamente hoy + 24hs a todos. El operador puede ajustar cada pedido individualmente abriendo el modal de detalle.

💡 Recomendado — cero fricción en el flujo masivo
✓ Cero fricción ✓ Escala bien ✓ Editable luego
⚙️ Nota técnica — Backend
El endpoint PATCH /order-header/bulk-status actualmente solo recibe { ids, status }.
Para soportar "Reprogramar" necesita aceptar también callStatus y callbackAt. ¿Podemos extender ese endpoint o preferís un endpoint separado?
